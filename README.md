##2026.04.08 Technical Decision

< #1. User 테이블 네이밍 유지 및 리팩토링 계획 >

초기 설계 단계에서 사용자 정보를 관리하는 테이블명을 **`user`** 로 정의하였다.<br>
프로젝트 진행 과정에서 **`user`** 는 데이터베이스 예약어와의 충돌 가능성이 있으며,<br> 
의미가 포괄적이어서 **실무에서는 명확한 네이밍으로 권장되지 않는다는 점을 확인** 하였다.<br>

현재 프로젝트는 회원가입, 로그인, 마이페이지, 관리자 기능까지 **`user`** 테이블을 기준으로 구현이 완료된 상태이다.<br>
또한 DTO, Service, Repository, Mapper 등 서비스 전반이 해당 테이블과 연결되어 있어,<br> 
단순한 테이블명 변경이 아닌 구조 전반에 영향을 주는 수정이 필요한 상황이다.<br>

이러한 점을 고려하여 현 시점에서는 프로젝트의 안정적인 기능 완성을 우선으로 판단하였고,<br> 
**테이블명 변경 없이 user를 유지** 하기로 결정하였다.<br>

향후 리팩토링 단계에서는 **테이블명을 users 또는 member로 변경** 하고,<br> 
이에 따라 도메인, DTO, Repository, Mapper 등 관련 구조를 일관성 있게 수정할 예정이다.<br> 
이후 전체 기능 테스트를 통해 시스템 안정성을 검증할 계획이다.<br>

이를 통해 예약어 충돌 가능성을 줄이고, 도메인 명확성과 유지보수성을 개선하는 것을 목표로 한다.
